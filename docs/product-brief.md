# Продуктовый бриф MBT Manager Assistant

## Назначение

MBT Manager Assistant — внутренний инструмент для менеджеров туристического агентства, которым нужны быстрые и единообразные ответы по турам, правилам и продажам.

Продукт в первую очередь создается для русскоязычных менеджеров MBT. Технический код, названия файлов, названия папок, TypeScript-типы, имена функций и названия documentation-файлов остаются на английском. Контент базы знаний для менеджеров по умолчанию пишется на русском.

Ассистент должен помогать менеджерам:

- Сравнивать варианты туров.
- Единообразно отвечать на вопросы команды.
- Быстро находить правила и операционные детали.
- Готовить понятные черновики ответов клиентам в нужном тоне агентства.

## Объем MVP

Активный MVP сейчас ограничен только направлением Mount Batur:

- Семейство рассветного треккинга на Батур.
- Варианты страницы рассветного треккинга на Батур, пока обозначенные временными anchor ID.
- Текущий статус Батура.
- Политики безопасности вулканических туров.
- Правила цены, доступности и бронирования, если они применимы к турам на Батур.
- Tone of voice продаж и Batur-разговоры для менеджеров.

Туры по Убуду, Улувату, обзорные туры по Бали и другие небатурные направления вынесены из активной базы знаний в архив. MVP должен отдавать приоритет точной архитектуре знаний по Батуру, ручной проверке рисков и валидируемым источникам, а не автоматизации. Интеграция OpenAI API намеренно не входит в этот начальный этап.

## Основные пользователи

- Русскоязычные менеджеры туристического агентства.
- Русскоязычные руководители продаж.
- Русскоязычные операционные координаторы.

## Языковая стратегия

- Технический код, названия файлов, названия папок, TypeScript-типы, имена функций и названия documentation-файлов остаются на английском.
- Контент базы знаний для менеджеров пишется на русском.
- Это включает карточки туров, tone of voice продаж, политики, примеры разговоров, примеры ответов клиентам и предупреждения для менеджеров.
- Английские ответы клиентам могут поддерживаться позже, но английский не должен становиться языком базы знаний по умолчанию.
- Если исходные данные доступны только на английском, нужно сохранить смысл источника и при необходимости отметить материал для русскоязычной проверки.

## Основные сценарии

- Подобрать подходящий вариант рассветного треккинга на Батур после проверки источников.
- Объяснить менеджеру, какие детали нужно уточнить до подтверждения клиенту.
- Отделить общие факты семейства Batur sunrise от фактов конкретного варианта.
- Требовать ручную проверку при вопросах про извержение, активность вулкана, пепел, дым, лаву, землетрясения, закрытие маршрута, текущую безопасность или доступность.
- Дать отделу продаж согласованные русскоязычные формулировки и предупреждения.

## Источники знаний

Активные знания хранятся в директории `knowledge/` с использованием предсказуемых markdown-шаблонов. Небатурные черновики хранятся в `_archive/non-batur/` и не считаются активной базой MVP. Контент должен быть удобен для проверки и обновления русскоязычной командой без инженерного контекста.

## Что не входит в объем

- Пока нет реализации OpenAI API.
- Пока нет UI.
- Пока нет клиентского чат-бота.
- Пока нет интеграции с системой бронирования.
- Пока нет live-синхронизации инвентаря, цен или доступности.
