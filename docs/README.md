# Документация проекта

Этот каталог — точка входа для будущего AI assistant. После чтения этих файлов ассистент должен уметь продолжить разработку с минимальным дополнительным контекстом.

## Reading order

1. [project-state.md](project-state.md)
2. [architecture.md](architecture.md)
3. [roadmap.md](roadmap.md)
4. [decision-log.md](decision-log.md)
5. [session-handoff.md](session-handoff.md)
6. [current-priorities.md](current-priorities.md)
7. [project-vision.md](project-vision.md)

## Как пользоваться

Сначала понять текущее состояние проекта, затем архитектуру и roadmap. После этого читать decision log и session handoff, чтобы не повторять старые решения. Current priorities показывает, что делать следующим. Project vision объясняет долгосрочную философию.

Важно: проект сейчас не про UI и не про OpenAI API. Главная задача — надежная Batur knowledge architecture и expert sales decision system для менеджеров MyBaliTrips.
