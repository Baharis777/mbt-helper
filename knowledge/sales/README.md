# База знаний по продажам

## Назначение

`knowledge/sales/` хранит постоянную методологию коммуникации и продаж MyBaliTrips. Это слой, который помогает ассистенту думать как опытный менеджер: понять клиента, выбрать подходящий продукт, задать нужные вопросы, обработать сомнение и довести диалог до следующего шага.

## Что читать первым

1. `sales-principles.md` - центральные принципы.
2. `communication-style.md` - стиль сообщений.
3. `qualification-strategy.md` - какие данные уточнять и когда.
4. `recommendation-strategy.md` - как выбирать подходящий продукт.
5. `conversation-patterns.md` - типовые этапы диалога.
6. `booking-strategy.md` - переход к бронированию.
7. `follow-up-strategy.md` - напоминания без давления.
8. `objection-handling.md` - работа с сомнениями.
9. `language-guidelines.md` - стиль по языкам.

## Связь с другими разделами

`knowledge/imports/` содержит временные сырьевые источники. Их нельзя использовать как постоянную базу знаний без переработки.

`knowledge/tours/` содержит продуктовые факты, варианты, ограничения и поля, требующие проверки.

`knowledge/rules/` содержит логику решений: как выбирать между продуктами и вариантами.

`knowledge/playbooks/` содержит прикладные сценарии работы менеджера для конкретного направления или продукта.

`knowledge/sales/` содержит постоянные принципы коммуникации и продаж, которые применимы шире одного тура.

## Важное правило

Знания по продажам не заменяют факты о турах. Если ассистенту нужно назвать цену, доступность, включения, безопасность или правила, он должен обращаться к продуктовой базе и актуальным проверкам.
