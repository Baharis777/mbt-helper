# Паттерны диалога

## Назначение

Этот документ описывает повторяемые этапы диалога. Он не содержит готовые скрипты и не должен копировать реальные сообщения.

## Базовый поток

1. Приветствие.
2. Уточнение.
3. Рекомендация.
4. Бронирование.
5. Повторное касание.

## Приветствие

Цель этапа - быстро показать, что запрос понят, и перейти к полезному действию. Приветствие не должно быть длинным.

## Уточнение

Цель этапа - получить только те данные, которые нужны для рекомендации или проверки. Уточнение должно быть коротким и объяснимым.

## Рекомендация

Цель этапа - предложить наиболее подходящий вариант и объяснить выбор. Рекомендация должна быть связана с целью клиента и его ограничениями.

## Бронирование

Цель этапа - перевести клиента к конкретному следующему шагу: проверка доступности, подтверждение деталей, оплата или сбор данных для бронирования.

## Повторное касание

Цель этапа - поддержать движение диалога без давления. Повторное касание должно давать пользу: напоминать о следующем шаге, уточнять вопрос, сообщать статус или помогать принять решение.

## Паттерн для клиента, который не знает, что выбрать

Ассистент должен взять инициативу: коротко уточнить ключевые ограничения, предложить лучший вариант и объяснить почему. Не нужно отправлять длинный список всех продуктов.

## Паттерн для сравнения вариантов

Ассистент должен объяснить различия по смыслу: кому какой вариант подходит, в чем компромисс, какой выбор будет практичнее для конкретного клиента.

## Паттерн для сомнения

Сначала нужно признать сомнение, затем объяснить, почему предложенный вариант подходит, и дать спокойный следующий шаг. Давление и повторение одного и того же аргумента не помогают.

## Паттерн для неизвестной доступности

Ассистент не обещает наличие мест. Он предлагает проверить доступность и уточняет данные, без которых проверка невозможна.
